# CardSlinger
**CardSlinger** – веб-приложение, представляющее собой интернет-магазин тематических товаров по настольной коллекционной карточной игре "*Heroes of Dariohn" (дальше ККИ).
К возможностям пользователя относятся просмотр каталога товаров с возможностью сортировки, поиск интересующего товара напрямую или через фильтры, просмотре
описания товаров, добавление и удаление товаров из корзины, список избранных товаров, покупка выбранных товаров.

Так же будет присутствовать админ-функции для собственно добавления и редактирования товаров, управления аккаунтами пользователей.

_"*Heroes of Dariohn" - это (технически) не существующая настольная игра, придуманная для демонстрации проекта сайта._

## Аудитория:
+ Игроки в ККИ "Heroes of Dariohn" и поклонники серии
+ Другие люди, чьим хобби являются настольные игры

## Ключевые функции:
+ Регистрация и авторизация
+ Просмотр каталога товаров
+ Поиск товаров по ключевым словам и с помощью филтров
+ Добавление и удаление товара из корзины
+ Покупка товаров, добавленных/удаление в корзину/желаемое
+ Админ-функции с добавление/удаление и редактирование пользователей и товаров

## Структура приложения:
1. Главная (`/`) – публичная, основная страница
2. Аутентификация (`/auth/*`) – регистрация и авторизация, восстановление пароля
3. Каталог (`/catalog`) – динамический список товаров
4. Страница товара (`/catalog/:id`) – динамическая
5. Личный кабинет (`/profile`) – приватный
6. Избранное (`/favorites`) – приватный
7. Поиск (`/search`) – с динамическими результатами
8. Админ-панель (`/admin`) – приватный

## Технологический стек
### Frontend: Чистый JavaScript, HTML, CSS
HTML для создания структуры веб-страниц. CSS для задания стилей и оформления положения элементов.
JavaScript обеспечивает интерактивность: он обрабатывает клики, управляет анимациями и валидирует данные в 
формах перед отправкой, нужен для отправки запросов к бэкенду и обновления содержимого страницы без её полной перезагрузки.

### Backend: Java
Java обеспечивает высокую надежность и безопасность благодаря строгой типизации и мощным инструментам защиты данных.
Легко справляется с высокими нагрузками и позволяет выстраивать сложную бизнес-логику.

### База данных: MySQL
MySQL — это проверенная временем реляционная база данных, гарантирующая целостность и сохранность информации. 
Она идеально подходит для структурированных данных и поддерживается практически любой серверной инфраструктурой.

### Хостинг: Byethost
Сервис выбран в связи с тем, что предлагает очень хороший бесплатный план без рекламы для пользователя, 
а так же предоставляет встроенную MySQL базу данных и phpMyAdmin для работы с базой данных.

## Интерфейс 
### Страница: Главная (`/`)
**Тип доступа:** публичный

**Основные блоки:**
1. **Шапка:** название магазина, кнопки для перехода в подменю каталога, аутентификации или личный кабинет(если авторизован)
2. **Информационный блок:** приветственное сообщение, краткое описание магазина и товаров, кнопка для перехода в каталог
3. **Блок "Новинки":** показывает карточки с недавно добавленными товарами
4. **Футер:** контактная иформация и информация о магазине
   
**Динамический контент:** карточки недавно добавленных товаров подгружаются из базы, отображение имени пользователя при авторизации

### Страница: Аутентификация (`/auth/*`)
**Тип доступа:** публичный

**Основные блоки:**
1. **Шапка:** название магазина, ссылка на главную страницу
2. **Форма регистрации:** имя пользователя, электронная почта, пароль, подтверждение пароля
3. **Форма входа:** поля логина и пароль, кнопка «Войти»
4. **Восстановление пароля:** поле с вводом электронной почты
5. **Страница для восстановления:** ввод нового пароля и подтверждение
   
**Динамический контент:** проверка правильности введённых данных с БД

### Страница: Каталог (`/catalog`)
**Тип доступа:** публичный

**Основные блоки:**
1. **Шапка:** название магазина, переход на главную, поиск, имя пользователя с переходом в профиль 
2. **Фильтры товаров:** фильтр по цене, типу товара, году выпуска, дополнению(e.g. оригинальное издание, второе издание и тд.)
3. **Сетка товаров:** карточки товаров с изображением, ценой, кнопкой добавления в корзину или в избранное
4. **Навинация:** переход на следующую, предыдущую или выбраннуб страницу в каталоге
5. **Футер:** контактная иформация и информация о магазине
   
**Динамический контент:** карточки товаров и результаты фильтрации подгружаются из базы

### Страница: Детальная страница товара (`/catalog/:id`)
**Тип доступа:** публичный

**Основные блоки:**
1. **Шапка:** название магазина, переход на главную, поиск, имя пользователя с переходом в профиль 
2. **Информация о товаре:** изображение, название, цена, детальное описание
3. **Кнопки действий:** добавить/удалить из избранного, добавить в корзину
   
**Динамический контент:** данные товара подгружаются из БД

### Страница: Личный кабинет (`/profile`)
**Тип доступа:** приватный

**Основные блоки:**
1. **Шапка:** название магазина, ссылка на главную страницу
2. **Информация о пользователе:** логин, почта
3. **Редактирование профиля:** изменение логина, возможность смены пароля через email
4. **Кнопка деавторизации**
   
**Динамический контент:** подгрузка данных пользователя из БД

### Страница: Избранное (`/favorites`)
**Тип доступа:** приватный

**Основные блоки:**
1. **Шапка:** название магазина, ссылка на главную страницу
2. **Список избранных товаров:** карточки помеченных как избарнные пользователем товаров
3. **Кнопки действий:** удалить из избранного, перейти к товару, добавить в корзину
   
**Динамический контент:** получение данных для карточек товаров из БД

### Страница: Поиск (`/search`)
**Тип доступа:** публичный

**Основные блоки:**
1. **Шапка:** название магазина, ссылка на главную страницу
2. **Поле поиска:** 
3. **Результаты поиска:** список товаров, подходящих по введённому значению и по фильтрам
4. Сообщение об отсутствии результатов, если никаких совпадений не найдено
   
**Динамический контент:** получение данных и БД и их отображение

### Страница: Админ-панель (`/admin`)
**Тип доступа:** приватный (только администратор)

**Основные блоки:**
1. **Шапка:** название магазина, ссылка на главную страницу, переход в меню товаров, переход в меню пользователей
2. **Список товаров:** таблица с наименованиями товаров и кнопками управления товарами
3. **Управление товарами:** возможность добавлять, удалять или изменять уже сущестующие товары
4. **Управление пользователями** настройка прав доступа, логина, просмотр пароля и почты
   
**Динамический контент:** получение информации о пользователях и товарах из БД

## Схема данных 
### User:
+ `id` - id пользователя
+ `username` - логин
+ `password` - пароль (сохраняется как хэш)
+ `is_admin` - определяет права пользователя

### Item:
+ `id` - id товара
+ `item_name` - наименование товара
+ `item_price` - цена товара
+ `item_category` - категория(тип) товара
+ `item_desc` - описание товара
+ `item_image_url` — ссылка на изображение товара
